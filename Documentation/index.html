Local Development Environment Setup

In order to kickstart the development of your web app with Ovent, the very first thing is to setup an Angular development environment.

Getting Started

Name Description
angular.json Used for configuration of project specific settings. You can add external styles and scripts, change the output folder, add assets, add environment files and more.
e2e Will be used for end-to-end tests to ensure functionality for users before deploying.
node_modules All external modules used are here. Do not mess with this folder, as it is auto-generated by using npm install.
package.json Contains all dependencies used for production and development.
src Contains all Angular Typescript code, assets and basically everything the end user will have access to.
tslint.json Angular-CLI includes an automatic Typescript-Linter, which can be configured with this file.
  • Node.js and NPM : If NodeJS is not installed then download it from https://nodejs.org/. NPM comes bundled with Node.js
  • angular-cli : You can install angular-cli by executing from your command window npm install -g @angular/cli. More details can be found here https://github.com/angular/angular-cli

NPM Packages

  • Before proceding you'll need to install npm packages. You can do this by running npm install from the root of your project to install all the necessary dependencies.

Development Server

  • Run ng serve for a dev server. Navigate to localhost:4200/. The app will autometically reload if you change any of the source files.

Building

  • Run ng build to build the project. The build artifacts will be stored in the dist/ directory. Use the --prod flag for a production build.

More Help

  • To get more help on the angular-cli use ng --help or go check out the Angular-CLI README.

Fonts

By default, the template loads Karla & Montserrat font from Google Web Font Services, you can change the font with the one that suits you best.

Font code can be found in the "assets/css/style.css" file:


@import url("https://fonts.googleapis.com/css2?family=Karla:ital,wght@0,400;0,700;1,400;1,700&family=Montserrat:wght@100;300;400;500;600;700;800;900&display=swap");

Setting Up Contact Form

To get started, the template provides you with a functional contact form using emailjs.com. We will be providing you with all the required steps to get started, and you can also read the documentation on https://www.emailjs.com/docs/examples/angular/.

Creating a new service

Navigate to emailjs.com, and create a new account. Once you're in your dashboard, you will see an option to create a new service. Click on the Add new service button to get started

Setting up the service you want (We will be using SMTP for this article)

Select the service that you need

Fetching the Service ID

Fill in the required data for this form, and make sure to save the service_id which we will be using in your application.

Creating an Email Template

We will not be covering how to create an email template because their documentation does this in much more detailed way. But you will need to create an Email template and save the template_id which we will be using in your application.

Fetching your User ID

Step 5 would be to go to your integrations, and this is by clicking on the integrations page into left sidebar, and finding your user_id which we will be using in your application.

Filling in the data in your application

Finally, navigate to contact.ts in your app and fill in the data as highlighted in the below image:

  1. Service id: service_id
  2. Template id: template_id
  3. User id: user_id

Once you have set up all the data correctly, you should have a fully functional contact form.
